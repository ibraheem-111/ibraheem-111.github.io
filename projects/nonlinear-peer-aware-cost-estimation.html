Nonlinear Peer-Aware Cost Estimation

Mobile robotics research using motion capture calibration and model predictive control

August 2025 - Ongoing | GHOST Lab, ASU

What I Built

  • Motion Capture System Integration: Calibrated a multi-camera motion capture system to detect real-time position of two rigid bodies with sub-centimeter accuracy.
  • ROS Robot Control Architecture: Developed C++ ROS nodes for real-time robot control loop and feedback collection at 100+ Hz.
  • Sensor Fusion Pipeline: Implemented Enhanced Kalman Filters (EKF) to fuse motion capture data and internal odometry, reducing sensor noise by ~40%.
  • Model Predictive Control: Designed and tuned an MPC controller that guides the robot's movement based on a nonlinear peer-aware cost function.
  • Experimentation Framework: Built automated experiment logging and data visualization tools for validating control algorithms.

Key Components:

  • Motion Capture: Optitrack system with 4 cameras, providing ground truth position at 100 Hz
  • EKF Sensor Fusion: Combines motion capture (high accuracy, low latency) with robot odometry (bias correction)
  • MPC Planner: Solves optimal control problem over 10-step prediction horizon, updates at 50 Hz
  • ROS2 Middleware: Nodes communicate via DDS, ensuring deterministic timing for control loops
  • Data Logger: Records all sensor data, control inputs, and robot state for post-experiment analysis

Key Engineering Decisions

  • EKF over Particle Filter: Chose EKF for its computational efficiency on edge devices and proven performance with Gaussian-distributed sensor noise from motion capture systems.
  • MPC over PID: Model Predictive Control provides superior handling of nonlinear cost functions and system constraints compared to traditional PID, enabling more sophisticated navigation behaviors.
  • ROS (not custom C++): Selected ROS for rapid prototyping, leveraging existing libraries for trajectory optimization and visualization tools. Decouples algorithm development from hardware control.
  • High-Frequency Control Loop (100+ Hz): Motion capture systems provide latency ~10ms; running control at 100 Hz ensures we can respond to rapid state changes and maintain stability.
  • Modular Node Architecture: Separated sensor fusion (EKF node), planning (MPC node), and control (hardware node) for independent testing and reusability across experiments.
  • Extensive Logging: Records all sensor readings and control outputs to enable offline analysis and machine learning-based parameter tuning later.

Results & Validation

  • Motion Capture Calibration: Achieved ±2cm accuracy across the 5m × 5m workspace using a 4-camera setup.
  • EKF Performance: Reduced sensor noise variance by 40% compared to raw motion capture data; response time to step disturbances: ~200ms.
  • MPC Tracking: Successfully tracks nonlinear cost function trajectories with <10cm steady-state error on a 3m × 3m path.
  • Real-time Performance: Control loop runs at stable 100 Hz with 2-3ms jitter; MPC computation time: ~40ms per cycle.
