Natural Language Controlled Pick and Place Robot

ROS2-based autonomous system using NLP and stereoscopic 3D vision for object manipulation

November 2025 - December 2025 | Arizona State University

What I Built

  • Natural Language Interface: NLP pipeline that parses user instructions (e.g., "Pick up the red block and place it on the left") into structured task commands for the robot.
  • 3D Object Detection: Stereoscopic camera system with custom computer vision algorithms to detect object dimensions, color, depth, and orientation in real-time (30 FPS).
  • ROS2 Action Orchestrator: Central orchestrator node that decomposes natural language tasks into atomic ROS2 actions (Pick, Place, Navigate) and delegates them to specialized nodes.
  • Multi-node Architecture: Separate ROS2 nodes for perception, planning, and arm control, enabling modularity and independent testing of each component.
  • Error Recovery: Implemented retry logic and visual feedback loops to handle perception failures and recover from dropped or misplaced objects.

Key Components:

  • NLP Module: Python-based parser using rule-based + lightweight ML to extract object attributes (color, size, destination) from free-form text.
  • Perception Node: Subscribes to stereo camera feeds, runs YOLO for object detection, stereo matching for depth, computes object 6D pose.
  • Orchestrator Node: Central ROS2 action server that coordinates Grasp, Place, and Navigate actions, handles inter-node communication via DDS.
  • Arm Control Node: Direct interface to UR5 robotic arm, receives grasp targets (3D position + orientation) and executes pick-place trajectory.
  • Feedback Loop: Re-scans scene after each action to verify task completion; triggers recovery if object not placed correctly.

Key Engineering Decisions

  • ROS2 over ROS1: Chose ROS2 for its modern DDS-based middleware, better real-time guarantees, and improved type safety (compared to ROS1's loosely-typed message passing).
  • ROS2 Actions over Services: Used ROS2 action framework for long-running tasks (grasp, place) to provide feedback and enable cancellation, rather than simple request-reply services.
  • Stereoscopic Vision over Single RGB-D: Dual-camera setup provides redundancy and more robust depth estimation; chose OpenCV stereo matching over commercial RGB-D sensors for cost and flexibility.
  • Modular NLP: Built a simple rule-based parser rather than training a large language model, reducing inference latency and allowing easy debugging of command misinterpretations.
  • YOLO for Detection: Selected YOLO v5 for real-time performance (30 FPS) with reasonable accuracy; lighter than Faster R-CNN and sufficient for structured block environments.
  • Fail-Safe Error Recovery: After every action, re-perceive the scene to verify success. If object is not where expected, log and alert rather than cascading failures.

Results & Validation

  • Natural Language Parsing Accuracy: 95% success rate on test set of 100+ unique command variations; fails on ambiguous or out-of-domain instructions.
  • Object Detection: 92% precision, 88% recall for block detection; achieves ~30 FPS processing speed on edge GPU (NVIDIA Jetson TX2).
  • Pick Success Rate: 87% successful grasps on first attempt; ~95% success after 1 retry (handles occasional slips).
  • Place Accuracy: Вұ5cm placement error; sufficient for workspace designated by color (e.g., "red zone", "blue zone").
